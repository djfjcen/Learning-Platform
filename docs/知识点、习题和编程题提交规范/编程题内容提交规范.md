# 编程题内容提交规范

> 适用对象：`exercise` 表  
> 生效范围：**仅编程题**（`PROGRAMMING`）  


---



## 一、禁止提交的内容（硬性）

| 禁止项 | 原因 |
|------|------|
| `type` 写成 `CODE`、`PROGRAM`、`CODING` 等 | 项目代码只识别 `PROGRAMMING` |
| `test_cases` 中使用 `output` | 当前后端 `TestCase` DTO 只读取 `expectedOutput` |
| 提交 `initial_code` | 初始代码使用统一默认模板，不需要每道题单独提交 |
| `answer` 只提交一种语言代码 | 标准答案必须同时包含 C、C++、Python 三种语言 |
| 提交非表字段作为独立字段 | 会增加数据录入工作量，非表字段请合并进 `description` |
| 缺失题目描述、标准答案代码、解析、测试用例 | 不符合内容审核和判题要求 |

---

## 二、`exercise` 表字段 → 提交字段映射（严格一致）

| `exercise` 表字段 | 是否必填 | 提交要求 |
|------------------|----------|----------|
| `knowledge_point_id` | ✅ | 数字，必须使用知识点 ID |
| `title` | ✅ | 题目标题，简短明确 |
| `type` | ✅ | 固定填写 `PROGRAMMING` |
| `difficulty` | ✅ | `EASY` / `MEDIUM` / `HARD` |
| `category` | ✅ | `BASIC` / `INTERVIEW` / `EXAM` |
| `answer` | ✅ | 标准答案代码，纯文本格式，必须包含 C、C++、Python 三种完整可运行代码 |
| `analysis` | ✅ | 详细解析，说明为什么这样做 |
| `description` | ✅ | 题干全文；输入说明、输出说明、样例、备注等都写在这里 |
| `initial_code` | ❌ | 不提交；由数据库录入或前端统一使用默认初始代码模板 |
| `test_cases` | ✅ | JSON 数组字符串，每项包含 `input` 和 `expectedOutput` |
| `time_limit` | ⚠️ | 可填，单位秒；不填时默认 `60` |
| `sort_order` | ⚠️ | 可填；不填时默认 `0` |
| `options` | ❌ | 编程题不使用 |
| `id` | ❌ | 自增生成，不提交 |
| `pass_rate` | ❌ | 后端统计，不提交 |
| `total_attempts` | ❌ | 后端统计，不提交 |
| `correct_attempts` | ❌ | 后端统计，不提交 |
| `create_time` / `update_time` | ❌ | 数据库自动维护，不提交 |
| `is_deleted` | ❌ | 逻辑删除字段，默认未删除，不提交 |

## 四、`answer` 三语言标准答案提交格式

提交时按固定标题分成 C、C++、Python 三段代码

````text
【C】
```c
完整可运行的 C 标准答案代码
```

【C++】
```cpp
完整可运行的 C++ 标准答案代码
```

【Python】
```python
完整可运行的 Python 标准答案代码
```
````

规则：

1. `【C】`、`【C++】`、`【Python】` 三个标题缺一不可。
2. 三种语言都必须是完整可运行代码，不能只写函数片段。
3. `answer` 只写答案代码，不写解题思路；解题思路写到 `analysis`。

## 五、默认初始代码说明

提交题目资料时不需要填写 `initial_code`。该字段由数据库录入或前端统一使用默认模板处理，避免每道题重复提交相同内容。

默认模板格式如下：

```json
{
  "c": "#include <stdio.h>\n\nint main() {\n    return 0;\n}",
  "c++": "#include <bits/stdc++.h>\nusing namespace std;\n\nint main() {\n    return 0;\n}",
  "python": ""
}
```

## 六、`test_cases` 测试用例格式

`test_cases` 必须提交为 JSON 数组字符串，每个对象只使用以下字段：

```json
[
  {
    "input": "3 5",
    "expectedOutput": "8"
  }
]
```

规则：

1. 必须使用 `expectedOutput`，不要写成 `output`。
2. 多行输入使用 `\n` 表示换行，例如：
   ```json
   { "input": "5\n1 2 3 4 5", "expectedOutput": "15" }
   ```
3. `expectedOutput` 只写程序应输出的内容，不写解释。
4. 每道编程题至少提供 3 个测试用例，覆盖正常情况和边界情况。

## 八、完整示例

````text
=== 知识点ID: 13
=== 知识点名称: 栈

## PROGRAMMING
题型: PROGRAMMING
难度: EASY
类别: BASIC
题目: 有效的括号
描述:
给定一个只包含 '('，')'，'{'，'}'，'['，']' 的字符串 s，判断括号是否有效。

输入说明：
输入一行字符串 s。

输出说明：
如果括号字符串有效，输出 true；否则输出 false。

样例输入：
()[]{}

样例输出：
true

备注：
字符串长度不超过 1000。

答案:
【C】
```c
#include <stdio.h>
#include <string.h>

int main() {
    char s[1005];
    char stack[1005];
    int top = -1;
    scanf("%s", s);

    for (int i = 0; s[i] != '\0'; i++) {
        char ch = s[i];
        if (ch == '(' || ch == '[' || ch == '{') {
            stack[++top] = ch;
        } else {
            if (top < 0) {
                printf("false");
                return 0;
            }
            char left = stack[top--];
            if ((ch == ')' && left != '(') || (ch == ']' && left != '[') || (ch == '}' && left != '{')) {
                printf("false");
                return 0;
            }
        }
    }

    printf(top == -1 ? "true" : "false");
    return 0;
}
```

【C++】
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    string s;
    cin >> s;
    stack<char> st;
    unordered_map<char, char> mp = {
        {')', '('},
        {']', '['},
        {'}', '{'}
    };

    for (char ch : s) {
        if (ch == '(' || ch == '[' || ch == '{') {
            st.push(ch);
        } else {
            if (st.empty() || st.top() != mp[ch]) {
                cout << "false";
                return 0;
            }
            st.pop();
        }
    }

    cout << (st.empty() ? "true" : "false");
    return 0;
}
```

【Python】
```python
s = input().strip()
stack = []
match = {')': '(', ']': '[', '}': '{'}

for ch in s:
    if ch in '([{':
        stack.append(ch)
    else:
        if not stack or stack[-1] != match[ch]:
            print('false')
            exit()
        stack.pop()

print('true' if not stack else 'false')
```
解析: 遍历字符串时，左括号入栈；遇到右括号时，若栈为空或栈顶括号类型不匹配，则字符串无效。遍历结束后栈为空，说明所有左括号都被正确匹配。
时间限制: 60
排序: 1
测试用例:
[
  { "input": "()[]{}", "expectedOutput": "true" },
  { "input": "(]", "expectedOutput": "false" },
  { "input": "([)]", "expectedOutput": "false" },
  { "input": "{[]}", "expectedOutput": "true" }
]
````
