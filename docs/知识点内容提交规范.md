# 知识点内容提交规范

> 用于指导各模块负责人按统一格式提交 OPERATION（操作原理）、COMPLEXITY（复杂度分析）、CODE（代码示例）资料

---

## 涉及的两张表

| 表名 | 用途 | 对应区块标记 |
|------|------|-------------|
| `knowledge_content` | 知识点内容（操作原理、复杂度分析） | `## OPERATION`、`## COMPLEXITY` |
| `code_example` | 代码示例 | `## CODE` |

## 不需要提交的内容

- **基本定义**：前端直接用 `knowledge_point.description` 字段，不需要重复提交
- **核心考点**：前端直接用 `knowledge_point.core_points` 字段，不需要重复提交
- **一级模块节点（ID 1~6）**：集合、线性表、树、图、查找、排序这 6 个是模块级节点，不需要填详情

---

## 提交格式

每个人负责几个知识点，按以下格式整理成一个**纯文本文件**。

```
=== 知识点ID: 11
=== 知识点名称: 顺序表

## OPERATION
标题: 顺序表插入操作
内容:
插入操作首先判断表是否已满，若已满则无法插入。
在第 i 个位置插入时，需要将第 i 个元素及之后的所有元素后移一位。
最好情况：在表尾插入，无需移动元素。
最坏情况：在表头插入，需移动 n 个元素。
平均移动次数为 n/2。

标题: 顺序表删除操作
内容:
删除第 i 个位置的元素后，需将第 i+1 个元素及之后的所有元素前移一位。
最好情况：删除表尾元素，无需移动。
最坏情况：删除表头元素，需移动 n-1 个元素。

## COMPLEXITY
标题: 时间复杂度与空间复杂度
内容:
随机访问：O(1)——通过下标直接定位。
插入操作：平均 O(n)，最好 O(1)（表尾），最坏 O(n)（表头）。
删除操作：平均 O(n)，最好 O(1)（表尾），最坏 O(n)（表头）。
空间复杂度：O(1)，操作在原地完成，不需要额外空间。

## CODE
标题: 顺序表插入操作实现
描述: 在动态数组实现的顺序表中，于第 pos 个位置插入元素 val
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

class SeqList {
    int* data;
    int capacity;
    int size;
public:
    SeqList(int cap) : capacity(cap), size(0) {
        data = new int[cap];
    }
    bool insert(int pos, int val) {
        if (pos < 1 || pos > size + 1 || size >= capacity)
            return false;
        for (int i = size; i >= pos; i--)
            data[i] = data[i - 1];
        data[pos - 1] = val;
        size++;
        return true;
    }
};

int main() {
    SeqList list(10);
    list.insert(1, 100);
    list.insert(2, 200);
    list.insert(2, 150);
    return 0;
}
```

## 规则说明

| 区块标记 | 数据库表 | 对应 `content_type` | 是否可多段 |
|----------|---------|---------------------|-----------|
| `## OPERATION` | `knowledge_content` | `OPERATION` | ✅ 可以有多个"标题+内容" |
| `## COMPLEXITY` | `knowledge_content` | `COMPLEXITY` | ✅ 可以有多个 |
| `## CODE` | `code_example` | — | ✅ 可以有多个"标题+描述+代码" |

| 字段 | 要求 |
|------|------|
| **标题** | 简短描述，不超过 200 字 |
| **内容** | 纯文本即可，自然分段，前端会自动换行处理 |
| **代码** | **必须完整可运行**，包含 `#include` 和 `main()` |
| **语言** | `C++` / `C` / `Python`，不填默认 C++ |
| **描述** | 可选，一两句话解释代码做什么 |
| **知识点ID** | 必须是数字（见下方对照表），不要填知识点名称 |

## 示例：完整提交文件

```
=== 知识点ID: 23
=== 知识点名称: 顺序查找

## OPERATION
标题: 基本顺序查找
内容:
从头到尾逐个比较关键字，找到则返回位置，遍历完未找到则返回查找失败。
可设置哨兵放在 0 号位置，省略每次循环中判断越界的比较，提高效率。
适用于无序表和链表，简单但效率低。

## COMPLEXITY
标题: 复杂度分析
内容:
平均查找长度 ASL_succ = (n+1)/2，ASL_fail = n+1。
时间复杂度 O(n)，空间复杂度 O(1)。

## CODE
标题: 带哨兵的顺序查找
描述: 用 data[0] 存目标值减少边界判断
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

int seqSearch(vector<int>& arr, int key) {
    arr[0] = key;
    int i = arr.size() - 1;
    while (arr[i] != key) i--;
    return i;
}

int main() {
    vector<int> arr = {0, 3, 7, 1, 9, 5};
    int pos = seqSearch(arr, 7);
    cout << (pos ? "Found at " + to_string(pos) : "Not found") << endl;
    return 0;
}

=== 知识点ID: 24
=== 知识点名称: 二分查找

## OPERATION
标题: 迭代二分查找
内容:
前提：线性表必须有序。
每次取区间中点比较，若相等则找到；若目标值小于中点值，搜索左半区；否则搜索右半区。
重复直到找到或区间为空。

## COMPLEXITY
标题: 复杂度分析
内容:
每次比较将搜索区间缩小一半，时间复杂度 O(log n)。
空间复杂度 O(1)（迭代版本）或 O(log n)（递归版本调用栈）。
仅适用于顺序存储的有序表，链表无法高效随机访问，不适合二分查找。

## CODE
标题: 迭代版二分查找
描述: 非递归写法，避免栈溢出
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

int binarySearch(vector<int>& arr, int key) {
    int left = 0, right = arr.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == key) return mid;
        else if (arr[mid] < key) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}

int main() {
    vector<int> arr = {1, 3, 5, 7, 9, 11};
    int pos = binarySearch(arr, 7);
    cout << (pos != -1 ? "Found at " + to_string(pos) : "Not found") << endl;
    return 0;
}
```
