# 视频讲解模块联调清单

## 模块范围

- 前端入口：`/video`
- 前端文件：`frontend/src/views/VideoPlayer/index.vue`、`frontend/src/api/modules/video.js`
- 后端入口：`/api/video`
- 数据表：`video`、`video_watch_record`
- 播放方案：第一版使用直链 MP4，不做上传、转码、HLS 或对象存储接入。

## 接口清单

| 方法 | 路径 | 用途 | 说明 |
| --- | --- | --- | --- |
| GET | `/api/video/list` | 查询视频列表 | 可传 `knowledgeId`，为空时查询全部视频 |
| GET | `/api/video/{id}` | 查询视频详情 | 视频不存在返回业务错误码 `1701` |
| GET | `/api/video/watch-record/{videoId}` | 查询观看记录 | 使用 `X-User-Id`，未传时后端默认用户 `1` |
| POST | `/api/video/watch-record` | 上报观看进度 | `watchProgress` 范围 `0-100`，大于等于 `95` 标记完成 |

## 联调步骤

1. 启动 MySQL 并确认 `video`、`video_watch_record` 表已创建。
2. 启动后端服务，确认 `/api/video/list` 能返回统一响应结构 `{ code, message, data }`。
3. 启动前端服务，打开 `/video`。
4. 在无视频数据时确认页面显示空状态。
5. 在 `video` 表写入一条直链 MP4 数据后刷新页面，确认列表、播放器和进度条显示正常。
6. 播放、暂停、播放结束时检查 `video_watch_record` 是否插入或更新同一用户同一视频的记录。
7. 刷新页面后重新进入同一视频，确认能按已保存进度恢复到接近位置。

## 已知边界

- 当前不提供视频上传和素材管理能力，视频数据由数据库脚本或人工插入。
- 当前用户识别沿用临时方案 `X-User-Id`，没有改动登录鉴权公共层。
- 视频观看行为尚未同步写入 `learning_record`，后续需要与学习统计模块 owner 对齐字段后再接入。
- 本模块只负责视频体验和联调收敛，不替其他模块修复各自核心业务问题。
