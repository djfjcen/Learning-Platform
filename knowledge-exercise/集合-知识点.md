=== 知识点ID: 7
=== 知识点名称: 集合表示法

## OPERATION
标题: 列举法表示集合
内容:
列举法（枚举法）是将集合中的所有元素一一列出，写在大括号内，元素之间用逗号分隔。
例如：A = {1, 2, 3, 4, 5} 表示由1到5这五个正整数组成的集合。
列举法适用于元素个数有限且较少的集合，优点是直观、一目了然。
注意：集合中元素不重复，{1, 2, 2, 3} 等价于 {1, 2, 3}；元素无序，{1, 2, 3} 与 {3, 1, 2} 表示同一集合。

标题: 描述法表示集合
内容:
描述法（特征性质法）是用集合中元素的公共特征来描述集合，一般形式为 {x | P(x)}，其中 P(x) 是元素 x 满足的条件。
例如：B = {x | x 是正偶数} 表示所有正偶数的集合。
描述法适用于元素个数无限或不宜逐一列举的集合，表达更简洁。
描述法的关键在于谓词 P(x) 必须是明确的判定条件，即对任意对象都能判断其是否属于该集合。

标题: 文氏图表示集合
内容:
文氏图（Venn Diagram）用封闭曲线（通常是圆或矩形）来直观表示集合及其关系。
矩形表示全集 U，圆表示各个子集，圆的重叠部分表示集合的交集。
文氏图不是严格的数学表示，但有助于理解集合之间的包含、相交、互斥等关系，常用于集合运算的可视化分析。

## COMPLEXITY
标题: 集合表示法的比较分析
内容:
列举法：适合有限小集合，存储空间 O(n)，查找需要遍历 O(n)。
描述法：适合无限集或大集合，不直接存储元素，无法直接枚举，需要通过判定条件逐个验证。
文氏图：仅用于可视化辅助理解，不作为程序存储结构。
在计算机中实现集合时，需要选择合适的数据结构（数组、链表、哈希表、位图等）来高效支持集合操作。

## CODE
标题: 列举法与描述法的程序表示
描述: 用数组和谓词函数分别模拟列举法和描述法
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

void enumerateSet(const vector<int>& A) {
    cout << "列举法: A = {";
    for (size_t i = 0; i < A.size(); i++) {
        cout << A[i];
        if (i + 1 < A.size()) cout << ", ";
    }
    cout << "}" << endl;
}

void descriptiveSet(int n) {
    cout << "描述法: B = {x | x 是1到" << n << "的正偶数} = {";
    bool first = true;
    for (int x = 2; x <= n; x += 2) {
        if (!first) cout << ", ";
        cout << x;
        first = false;
    }
    cout << "}" << endl;
}

int main() {
    vector<int> A = {1, 2, 3, 4, 5};
    enumerateSet(A);
    descriptiveSet(10);
    return 0;
}

=== 知识点ID: 8
=== 知识点名称: 元素隶属关系

## OPERATION
标题: 属于与不属于
内容:
若元素 a 是集合 A 中的成员，则记作 a ∈ A，读作"a 属于 A"。
若元素 a 不是集合 A 中的成员，则记作 a ∉ A，读作"a 不属于 A"。
隶属关系是集合论中最基本的关系，是定义集合运算和推导集合性质的基础。
在程序中，隶属关系对应"查找"操作——判断某个值是否在集合中存在。

标题: 集合的包含关系
内容:
若集合 A 的每个元素都属于集合 B，则称 A 是 B 的子集，记作 A ⊆ B。
若 A ⊆ B 且 B 中至少有一个元素不属于 A，则称 A 是 B 的真子集，记作 A ⊂ B。
空集 ∅ 是任何集合的子集，是任何非空集合的真子集。
任何集合是其自身的子集（自反性），包含关系具有传递性。

标题: 集合的相等
内容:
若 A ⊆ B 且 B ⊆ A，则 A = B，即两个集合包含完全相同的元素。
集合相等只取决于元素本身，与元素的排列顺序无关。
判断两个集合相等的方法：逐一验证 A 中每个元素都属于 B，且 B 中每个元素都属于 A。

## COMPLEXITY
标题: 隶属判断的复杂度
内容:
基于数组的无序集合：判断 a ∈ A 需要遍历，时间复杂度 O(n)。
基于有序数组：可用二分查找，时间复杂度 O(log n)。
基于哈希表：平均时间复杂度 O(1)，最坏 O(n)（哈希冲突严重时）。
基于位图（元素为小整数）：O(1)，直接按位判断。
子集判断 A ⊆ B：朴素方法对 A 中每个元素在 B 中查找，O(|A|·|B|)；使用哈希集合可优化到 O(|A|+|B|)。

## CODE
标题: 隶属关系与子集判断
描述: 判断元素是否属于集合，以及判断子集关系
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

bool isMember(const set<int>& S, int x) {
    return S.count(x) > 0;
}

bool isSubset(const set<int>& A, const set<int>& B) {
    for (int x : A) {
        if (!B.count(x)) return false;
    }
    return true;
}

int main() {
    set<int> A = {1, 2, 3};
    set<int> B = {1, 2, 3, 4, 5};

    cout << "3 ∈ A: " << (isMember(A, 3) ? "true" : "false") << endl;
    cout << "4 ∈ A: " << (isMember(A, 4) ? "true" : "false") << endl;
    cout << "A ⊆ B: " << (isSubset(A, B) ? "true" : "false") << endl;
    cout << "B ⊆ A: " << (isSubset(B, A) ? "true" : "false") << endl;
    return 0;
}

=== 知识点ID: 9
=== 知识点名称: 集合运算

## OPERATION
标题: 并集运算
内容:
集合 A 与 B 的并集记作 A ∪ B = {x | x ∈ A 或 x ∈ B}，即由属于 A 或属于 B 的所有元素组成的集合。
并集的性质：A ∪ A = A（幂等律），A ∪ ∅ = A（同一律），A ∪ U = U（U 为全集，零律）。
在程序中，并集操作对应合并两个集合并去重。

标题: 交集运算
内容:
集合 A 与 B 的交集记作 A ∩ B = {x | x ∈ A 且 x ∈ B}，即由同时属于 A 和 B 的所有元素组成的集合。
交集的性质：A ∩ A = A（幂等律），A ∩ ∅ = ∅（零律），A ∩ U = A（同一律）。
若 A ∩ B = ∅，则称 A 与 B 互不相交（互斥）。

标题: 差集运算
内容:
集合 A 与 B 的差集记作 A - B = {x | x ∈ A 且 x ∉ B}，即由属于 A 但不属于 B 的元素组成的集合。
差集不满足交换律：A - B ≠ B - A（一般情况下）。
补集是差集的特殊情况：A 的补集 Ā = U - A，其中 U 是全集。

标题: 对称差运算
内容:
集合 A 与 B 的对称差记作 A △ B = (A - B) ∪ (B - A) = (A ∪ B) - (A ∩ B)，即属于 A 或 B 但不同时属于两者的元素。
对称差满足交换律：A △ B = B △ A，满足结合律：(A △ B) △ C = A △ (B △ C)。
对称差在集合论中的地位类似于整数加法。

标题: 笛卡尔积运算
内容:
集合 A 与 B 的笛卡尔积记作 A × B = {(a, b) | a ∈ A, b ∈ B}，即所有有序对组成的集合。
|A × B| = |A| × |B|，笛卡尔积的基数等于两个集合基数的乘积。
笛卡尔积不满足交换律：A × B ≠ B × A（除非 A = B 或其中之一为空集）。

## COMPLEXITY
标题: 集合运算的复杂度分析
内容:
设 |A| = m，|B| = n，使用哈希集合实现：
并集 A ∪ B：遍历 A 和 B 所有元素插入结果集，O(m + n)。
交集 A ∩ B：遍历较小集合，在较大集合中查找，O(min(m, n)) 平均。
差集 A - B：遍历 A，对每个元素在 B 中查找，O(m) 平均。
对称差：等价于并集减去交集，O(m + n)。
笛卡尔积：必须枚举所有有序对，O(m × n)，结果集大小为 m × n。
使用有序数组实现时，并集和交集可用归并思想，O(m + n)。

## CODE
标题: 集合基本运算实现
描述: 实现并集、交集、差集、对称差
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

set<int> setUnion(const set<int>& A, const set<int>& B) {
    set<int> result = A;
    result.insert(B.begin(), B.end());
    return result;
}

set<int> setIntersection(const set<int>& A, const set<int>& B) {
    set<int> result;
    for (int x : A) {
        if (B.count(x)) result.insert(x);
    }
    return result;
}

set<int> setDifference(const set<int>& A, const set<int>& B) {
    set<int> result;
    for (int x : A) {
        if (!B.count(x)) result.insert(x);
    }
    return result;
}

set<int> setSymmetricDiff(const set<int>& A, const set<int>& B) {
    auto diff1 = setDifference(A, B);
    auto diff2 = setDifference(B, A);
    return setUnion(diff1, diff2);
}

void printSet(const string& name, const set<int>& S) {
    cout << name << " = {";
    bool first = true;
    for (int x : S) {
        if (!first) cout << ", ";
        cout << x;
        first = false;
    }
    cout << "}" << endl;
}

int main() {
    set<int> A = {1, 2, 3, 4};
    set<int> B = {3, 4, 5, 6};
    printSet("A", A);
    printSet("B", B);
    printSet("A ∪ B", setUnion(A, B));
    printSet("A ∩ B", setIntersection(A, B));
    printSet("A - B", setDifference(A, B));
    printSet("A △ B", setSymmetricDiff(A, B));
    return 0;
}

=== 知识点ID: 10
=== 知识点名称: 集合存储实现

## OPERATION
标题: 顺序存储实现
内容:
用数组（顺序表）存储集合元素，元素在连续内存空间中依次存放。
无序数组：插入 O(1)（尾部追加），查找 O(n)，删除 O(n)（需移动元素）。
有序数组：插入 O(n)（需移动），查找 O(log n)（二分查找），删除 O(n)。
顺序存储适合元素个数已知且变化不大的场景，支持随机访问，但不便于动态扩容。

标题: 链式存储实现
内容:
用链表存储集合元素，每个节点包含数据域和指针域。
无序链表：插入 O(1)（头部插入），查找 O(n)，删除 O(n)（需先查找）。
有序链表：插入 O(n)（需找位置），查找 O(n)，删除 O(n)。
链式存储适合元素个数变化频繁的场景，动态分配内存，但不支持随机访问。

标题: 哈希表存储实现
内容:
用哈希表存储集合元素，通过哈希函数将元素映射到桶（bucket）中。
插入：计算哈希值，放入对应桶，平均 O(1)。
查找：计算哈希值，在对应桶中查找，平均 O(1)。
删除：查找后删除，平均 O(1)。
冲突处理方法：开放寻址法（线性探测、二次探测）和链地址法（拉链法）。
哈希表是集合操作最高效的实现方式，但不支持有序遍历，且最坏情况退化为 O(n)。

标题: 位图存储实现
内容:
当集合元素为有限范围内的小整数时，可用位图（bitmap）存储。
每个整数对应一个比特位，1 表示属于集合，0 表示不属于。
插入/查找/删除：直接操作对应比特位，O(1)。
空间复杂度：O(U/8) 字节，U 为元素取值范围上限。
位图适合稠密小整数集合，内存效率极高，但不适合稀疏集合或非整数元素。

## COMPLEXITY
标题: 集合存储实现的复杂度比较
内容:
| 操作 | 无序数组 | 有序数组 | 链表 | 哈希表 | 位图 |
|------|---------|---------|------|--------|------|
| 插入 | O(1) | O(n) | O(1) | O(1)* | O(1) |
| 查找 | O(n) | O(log n) | O(n) | O(1)* | O(1) |
| 删除 | O(n) | O(n) | O(n) | O(1)* | O(1) |
| 空间 | O(n) | O(n) | O(n) | O(n) | O(U) |
*哈希表为平均复杂度，最坏 O(n)
选择建议：频繁查找选哈希表，需要有序选有序数组/平衡树，小整数集合选位图，频繁插入删除选链表。

## CODE
标题: 哈希集合实现
描述: 用哈希表实现集合的插入、查找、删除操作
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

class HashSet {
    vector<list<int>> table;
    int capacity;
    size_t hash(int x) { return ((size_t)x * 2654435761u) % capacity; }
public:
    HashSet(int cap = 1009) : capacity(cap), table(cap) {}
    bool insert(int x) {
        size_t idx = hash(x);
        for (int v : table[idx]) if (v == x) return false;
        table[idx].push_back(x);
        return true;
    }
    bool contains(int x) {
        size_t idx = hash(x);
        for (int v : table[idx]) if (v == x) return true;
        return false;
    }
    bool remove(int x) {
        size_t idx = hash(x);
        auto& bucket = table[idx];
        for (auto it = bucket.begin(); it != bucket.end(); ++it) {
            if (*it == x) { bucket.erase(it); return true; }
        }
        return false;
    }
    void printAll() {
        cout << "{";
        bool first = true;
        for (auto& bucket : table)
            for (int v : bucket) {
                if (!first) cout << ", ";
                cout << v;
                first = false;
            }
        cout << "}" << endl;
    }
};

int main() {
    HashSet s;
    s.insert(10); s.insert(20); s.insert(30);
    cout << "contains(20): " << (s.contains(20) ? "true" : "false") << endl;
    cout << "contains(99): " << (s.contains(99) ? "true" : "false") << endl;
    s.remove(20);
    cout << "after remove(20): "; s.printAll();
    return 0;
}
