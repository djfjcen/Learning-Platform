=== 知识点ID: 7
=== 知识点名称: 集合表示法

## OPERATION
标题: 列举法表示集合
内容:
列举法是将集合中的所有元素一一列出来，用花括号{}括起来，元素之间用逗号分隔。
例如：A = {1, 2, 3, 4, 5} 表示由1到5这五个整数组成的集合。
列举法的优点是直观清晰，一目了然地看到集合包含哪些元素。
缺点是当集合元素很多或无限时，无法用列举法完整表示。
对于有规律的无限集，可以使用省略号，如自然数集 N = {0, 1, 2, 3, ...}。

标题: 描述法（构造式）表示集合
内容:
描述法通过描述集合中元素的共同特征来表示集合。
一般形式为：{x | P(x)} 或 {x : P(x)}，读作"所有满足条件P的x构成的集合"。
其中竖线左边x代表元素的一般形式，右边P(x)是元素满足的性质或条件。
例如：B = {x | x是正偶数} = {2, 4, 6, 8, ...}
再如：C = {x² | x ∈ N, x < 5} = {0, 1, 4, 9, 16}
描述法可以表示有限集和无限集，是更通用、更数学化的表示方法。

## COMPLEXITY
标题: 集合基本操作复杂度分析
内容:
判断元素是否属于集合（∈运算）：取决于底层存储结构。
使用数组/链表存储时，查找时间复杂度为 O(n)。
使用哈希表存储时，平均查找时间复杂度可达到 O(1)，最坏情况 O(n)。
集合的并、交、差等运算的时间复杂度同样依赖于实现方式：
基于哈希表的实现通常为 O(m+n)，m、n为两集合元素个数。
基于有序列表的实现可以利用双指针优化至 O(m+n)。

## CODE
标题: 集合的基本操作实现（基于C++ STL set）
描述: 演示集合的创建、插入、删除、查找和遍历操作
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

int main() {
    // 创建空集合
    set<int> s;

    // 插入元素
    s.insert(3);
    s.insert(1);
    s.insert(4);
    s.insert(1);   // 重复元素自动忽略

    cout << "集合元素: ";
    for (int x : s)
        cout << x << " ";
    cout << endl;

    // 判断元素是否存在（查找）
    int target = 3;
    if (s.find(target) != s.end())
        cout << target << " 属于该集合" << endl;
    else
        cout << target << " 不属于该集合" << endl;

    // 删除元素
    s.erase(2);   // 删除不存在的元素安全无影响
    s.erase(3);

    cout << "删除后集合大小: " << s.size() << endl;
    return 0;
}

=== 知识点ID: 8
=== 知识点名称: 元素隶属关系

## OPERATION
标题: 元素与集合的关系判定
内容:
元素与集合之间存在两种基本关系：属于（∈）和不属于（∉）。
若a是集合A中的元素，记作 a ∈ A，读作"a属于A"。
若a不是集合A中的元素，记作 a ∉ A，读作"a不属于A"。
这种关系是确定的：对于任意元素a和任意集合A，a ∈ A 和 a ∉ A 有且仅有一个成立。
在编程中，隶属关系的判定对应于数据结构中的查找操作。

标题: 子集与真子集
内容:
若集合B的所有元素都是集合A的元素，则称B是A的子集，记作 B ⊆ A。
任意集合都是自身的子集，即 A ⊆ A。
空集 ∅ 是任何集合的子集。
若 B ⊆ A 且 B ≠ A（即A中存在不属于B的元素），则称B是A的真子集，记作 B ⊂ A。
子集关系具有传递性：若 A ⊆ B 且 B ⊆ C，则 A ⊆ C。
n个元素的集合共有 2^n 个子集（包括空集和自身）。

## COMPLEXITY
标题: 隶属关系与子集判定复杂度
内容:
单元素隶属判定（∈）：哈希表 O(1) 平均，数组/链表 O(n)。
子集判定（⊆）：设|A|=m，|B|=n，
- 哈希表实现：O(m) 预处理 + O(n) 查找 = O(m+n)
- 排序后双指针：O(m log m + n log n + m + n)
幂集生成（求所有子集）：2^n 个子集，时间复杂度 O(n × 2^n)，
空间复杂度 O(2^n)（输出规模决定）。

## CODE
标题: 子集判定算法实现
描述: 判断集合B是否为集合A的子集
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

bool isSubset(const vector<int>& A, const vector<int>& B) {
    unordered_set<int> setA(A.begin(), A.end());
    for (int x : B) {
        if (setA.find(x) == setA.end())
            return false;
    }
    return true;
}

int main() {
    vector<int> A = {1, 2, 3, 4, 5};
    vector<int> B = {2, 3, 5};
    vector<int> C = {2, 3, 6};

    cout << boolalpha;
    cout << "{2,3,5} 是 {1,2,3,4,5} 的子集? " << isSubset(A, B) << endl;
    cout << "{2,3,6} 是 {1,2,3,4,5} 的子集? " << isSubset(A, C) << endl;
    return 0;
}

=== 知识点ID: 9
=== 知识点名称: 集合运算

## OPERATION
标题: 并集运算 (∪)
内容:
两个集合A和B的并集 A ∪ B 是由所有属于A或属于B（或同时属于两者）的元素构成的集合。
形式化定义：A ∪ B = {x | x ∈ A 或 x ∈ B}
并集运算满足交换律：A ∪ B = B ∪ A
并集运算满足结合律：(A ∪ B) ∪ C = A ∪ (B ∪ C)
同一律：A ∪ ∅ = A
幂等律：A ∪ A = A

标题: 交集运算 (∩)
内容:
两个集合A和B的交集 A ∩ B 是由所有既属于A又属于B的元素构成的集合。
形式化定义：A ∩ B = {x | x ∈ A 且 x ∈ B}
交集运算同样满足交换律和结合律。
分配律：A ∩ (B ∪ C) = (A ∩ B) ∪ (A ∩ C)
若 A ∩ B = ∅，则称A和B互不相交（互斥）。

标题: 差集与补集运算
内容:
A与B的差集 A - B（或 A \ B）是由所有属于A但不属于B的元素构成的集合。
形式化定义：A - B = {x | x ∈ A 且 x ∉ B}
注意：差集运算不满足交换律，一般情况下 A - B ≠ B - A。
补集是在全集U下定义的：A的补集 Ā = U - A = {x | x ∈ U 且 x ∉ A}
德摩根定律连接了并交补三种运算：
(A ∪ B) 的补集 = Ā ∩ B̄
(A ∩ B) 的补集 = Ā ∪ B̄

## COMPLEXITY
标题: 集合运算复杂度总结
内容:
假设两个集合大小分别为 m 和 n：

并集 A ∪ B：遍历两个集合去重，O(m + n)
交集 A ∩ B：较小集合逐元素在大集合中查找，O(min(m,n) × 查找代价)
- 哈希表：O(min(m,n)) 平均
- 有序数组双指针：O(m + n)

差集 A - B：类似交集逻辑，O(m + n) 或 O(m) 哈希表
对称差集 A △ B = (A-B) ∪ (B-A)：O(m + n)

空间复杂度：结果集合的大小决定了额外空间，最坏 O(m+n)。

## CODE
标题: 集合运算完整实现
描述: 实现并集、交集、差集、对称差集四种基本运算
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

vector<int> setUnion(const vector<int>& A, const vector<int>& B) {
    unordered_set<int> result(A.begin(), A.end());
    for (int x : B) result.insert(x);
    return vector<int>(result.begin(), result.end());
}

vector<int> setIntersection(const vector<int>& A, const vector<int>& B) {
    unordered_set<int> setA(A.begin(), A.end());
    vector<int> result;
    for (int x : B)
        if (setA.count(x))
            result.push_back(x);
    return result;
}

vector<int> setDifference(const vector<int>& A, const vector<int>& B) {
    unordered_set<int> setB(B.begin(), B.end());
    vector<int> result;
    for (int x : A)
        if (!setB.count(x))
            result.push_back(x);
    return result;
}

vector<int> symmetricDifference(const vector<int>& A, const vector<int>& B) {
    auto ab = setDifference(A, B);
    auto ba = setDifference(B, A);
    return setUnion(ab, ba);
}

void printSet(const string& name, const vector<int>& s) {
    cout << name << " = {";
    for (size_t i = 0; i < s.size(); i++) {
        cout << s[i];
        if (i < s.size() - 1) cout << ", ";
    }
    cout << "}" << endl;
}

int main() {
    vector<int> A = {1, 2, 3, 4};
    vector<int> B = {3, 4, 5, 6};

    printSet("A", A);
    printSet("B", B);
    printSet("A ∪ B", setUnion(A, B));
    printSet("A ∩ B", setIntersection(A, B));
    printSet("A - B", setDifference(A, B));
    printSet("A △ B", symmetricDifference(A, B));
    return 0;
}

=== 知识点ID: 10
=== 知识点名称: 集合存储实现

## OPERATION
标题: 位图（位向量）存储
内容:
位图是一种用二进制位来表示集合元素存在与否的高效存储方式。
适用于元素范围已知且较小的场景，如整数集合 {0, 1, ..., n-1}。
每个元素对应一个bit位：第i位为1表示i属于集合，为0表示不属于。
一个长度为n的位图可以用 n/8 个字节存储，空间效率极高。
优点：空间紧凑，并集/交集/差集可用位运算快速完成（O(n/w)，w为字长）。
缺点：只能存储离散的、可枚举的元素类型；大范围稀疏集合浪费空间。

标题: 哈希表存储
内容:
哈希表是实现动态集合的最常用方式之一。
通过哈希函数将元素映射到桶（bucket）中，支持平均 O(1) 的插入、删除、查找。
支持任意可哈希类型的元素，不需要预先知道元素范围。
需要处理哈希冲突（开放寻址法 / 链地址法）。
C++ STL 的 unordered_set 底层就是哈希表实现。
Java 的 HashSet 同样基于 HashMap 实现。
适合元素数量不确定、需要频繁查找的场景。

标题: 平衡搜索树存储
内容:
平衡搜索树（如红黑树、AVL树）可以实现有序集合。
元素按关键字排序存储，支持 O(log n) 的插入、删除、查找。
额外支持范围查询、前驱后继、最大最小值等操作。
C++ STL 的 set / map 底层通常采用红黑树实现。
Java 的 TreeSet 基于 TreeMap（红黑树）。
相比哈希表，平衡树占用更多内存但提供有序性保证。

## COMPLEXITY
标题: 三种存储方式的复杂度对比
内容:

| 操作       | 位图     | 哈希表(平均) | 平衡搜索树 |
|-----------|---------|-------------|-----------|
| 查找      | O(1)    | O(1)        | O(log n)   |
| 插入      | O(1)    | O(1)        | O(log n)   |
| 删除      | O(1)    | O(1)        | O(log n)   |
| 并集      | O(n/w)  | O(n)        | O((m+n)log(m+n)) |
| 最小/最大 | O(n/w)  | O(n)        | O(log n)   |
| 范围查询  | O(n)    | O(n)        | O(k+log n) |
| 空间开销  | O(U)    | O(n)        | O(n)       |

注：w为机器字长，U为值域大小，k为范围内元素数，n为集合大小。
位图的 O(U) 空间指值域大小而非实际元素数。

## CODE
标题: 位图集合实现
描述: 用整型数组模拟位图，支持基本的集合操作
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

class BitSet {
private:
    vector<uint32_t> bits;
    int size;
public:
    BitSet(int n) : size(n), bits((n + 31) / 32, 0) {}

    void add(int x) {
        if (x >= 0 && x < size)
            bits[x >> 5] |= (1u << (x & 31));
    }

    void remove(int x) {
        if (x >= 0 && x < size)
            bits[x >> 5] &= ~(1u << (x & 31));
    }

    bool contains(int x) const {
        if (x < 0 || x >= size) return false;
        return (bits[x >> 5] & (1u << (x & 31))) != 0;
    }

    BitSet unionWith(const BitSet& other) const {
        BitSet result(max(size, other.size));
        for (size_t i = 0; i < result.bits.size(); i++) {
            uint32_t a = (i < bits.size()) ? bits[i] : 0;
            uint32_t b = (i < other.bits.size()) ? other.bits[i] : 0;
            result.bits[i] = a | b;
        }
        return result;
    }

    void print() const {
        cout << "{";
        bool first = true;
        for (int i = 0; i < size; i++) {
            if (contains(i)) {
                if (!first) cout << ", ";
                cout << i;
                first = false;
            }
        }
        cout << "}" << endl;
    }
};

int main() {
    BitSet A(20);
    A.add(1); A.add(3); A.add(5); A.add(7); A.add(9);

    BitSet B(20);
    B.add(3); B.add(6); B.add(7); B.add(10);

    cout << "A = "; A.print();
    cout << "B = "; B.print();

    BitSet C = A.unionWith(B);
    cout << "A ∪ B = "; C.print();

    cout << "5 属于 A? " << (A.contains(5) ? "是" : "否") << endl;
    cout << "8 属于 A? " << (A.contains(8) ? "是" : "否") << endl;

    A.remove(5);
    cout << "删除5后 A = "; A.print();
    return 0;
}
