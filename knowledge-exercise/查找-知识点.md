=== 知识点ID: 23
=== 知识点名称: 顺序查找

## OPERATION
标题: 基本顺序查找
内容:
顺序查找从表的一端开始，依次把每个元素的关键字与目标关键字比较。
若某次比较相等，则查找成功并返回该元素的位置；若所有元素都比较完仍不相等，则查找失败。
顺序查找不要求数据有序，也不要求顺序存储，因此数组、链表、无序表都可以使用。

标题: 哨兵顺序查找
内容:
哨兵法常用于一维数组：把待查找的关键字放在 0 号位置，然后从表尾向前比较。
循环只需判断当前元素是否等于关键字，不必每次同时判断下标是否越界。
该方法不改变顺序查找的数量级，但能减少循环中的判断次数，适合讲解算法优化思想。

## COMPLEXITY
标题: 顺序查找复杂度
内容:
查找成功时，若每个元素被查到的概率相同，平均查找长度 ASL_success = (n + 1) / 2。
查找失败时，需要比较全部 n 个元素；若采用哨兵从尾向前查找，通常还会比较到 0 号哨兵。
时间复杂度为 O(n)，空间复杂度为 O(1)。
顺序查找简单、适用面广，但在大规模数据中效率较低。

## CODE
标题: 带哨兵的顺序查找
描述: data[0] 作为哨兵，返回目标在 1 到 n 范围内的位置，返回 0 表示未找到
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

int sequentialSearch(vector<int>& data, int key) {
    data[0] = key;
    int i = (int)data.size() - 1;
    while (data[i] != key) {
        i--;
    }
    return i;
}

int main() {
    vector<int> data = {0, 18, 27, 9, 45, 36};
    int key = 45;
    int pos = sequentialSearch(data, key);
    if (pos == 0) {
        cout << "Not Found" << endl;
    } else {
        cout << "Found at position " << pos << endl;
    }
    return 0;
}

=== 知识点ID: 24
=== 知识点名称: 二分查找

## OPERATION
标题: 二分查找过程
内容:
二分查找又称折半查找，前提是线性表按关键字有序，并且能够随机访问。
每次取当前查找区间的中间位置，与目标关键字比较。
若相等，则查找成功；若目标更小，则继续在左半区查找；若目标更大，则继续在右半区查找。
当左边界大于右边界时，说明查找区间为空，查找失败。

标题: 二分查找边界
内容:
实现二分查找时要明确区间含义，常见写法是闭区间 [left, right]。
循环条件使用 left <= right，更新边界时写成 left = mid + 1 或 right = mid - 1。
计算 mid 时推荐使用 left + (right - left) / 2，避免 left + right 在极大数据下溢出。

## COMPLEXITY
标题: 二分查找复杂度
内容:
每次比较后查找区间约缩小一半，因此最坏比较次数约为 floor(log2 n) + 1。
时间复杂度为 O(log n)。
迭代版本空间复杂度为 O(1)，递归版本空间复杂度为 O(log n)。
二分查找适用于有序数组，不适合普通链表，因为链表无法 O(1) 访问中间元素。

## CODE
标题: 迭代二分查找
描述: 在升序数组中查找目标值，返回下标，找不到返回 -1
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

int binarySearch(const vector<int>& a, int key) {
    int left = 0, right = (int)a.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (a[mid] == key) return mid;
        if (a[mid] < key) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return -1;
}

int main() {
    vector<int> a = {3, 8, 15, 22, 37, 49, 60};
    int key = 37;
    cout << binarySearch(a, key) << endl;
    return 0;
}

=== 知识点ID: 25
=== 知识点名称: 哈希表

## OPERATION
标题: 哈希查找思想
内容:
哈希表通过哈希函数把关键字映射到表中的存储位置。
查找时先计算目标关键字的哈希地址，再到对应位置比较关键字。
理想情况下，关键字能均匀分布到各个位置，查找不需要逐个扫描整个表。

标题: 冲突处理
内容:
不同关键字可能得到相同哈希地址，这种情况称为冲突。
常见冲突处理方法包括开放定址法和链地址法。
开放定址法在冲突后按一定探测序列寻找下一个空位置，例如线性探测、二次探测。
链地址法把同一个地址上的元素组织成链表或其他结构，查找时只在对应桶内比较。

## COMPLEXITY
标题: 哈希表复杂度
内容:
在哈希函数设计合理、负载因子适当的情况下，插入、删除和查找的平均时间复杂度为 O(1)。
若大量关键字集中到少数桶中，最坏时间复杂度可能退化为 O(n)。
空间复杂度为 O(m + n)，其中 m 是哈希表桶数，n 是元素个数。
哈希表适合等值查找，不适合天然支持有序遍历和范围查询。

## CODE
标题: 链地址法哈希表
描述: 用 vector<list<int>> 实现整数集合的插入与查找
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

class HashTable {
private:
    vector<list<int>> table;

    int hashValue(int key) const {
        return ((key % (int)table.size()) + (int)table.size()) % (int)table.size();
    }

public:
    explicit HashTable(int size = 11) : table(size) {}

    void insert(int key) {
        int index = hashValue(key);
        for (int value : table[index]) {
            if (value == key) return;
        }
        table[index].push_back(key);
    }

    bool contains(int key) const {
        int index = hashValue(key);
        for (int value : table[index]) {
            if (value == key) return true;
        }
        return false;
    }
};

int main() {
    HashTable ht;
    ht.insert(15);
    ht.insert(26);
    ht.insert(37);
    cout << (ht.contains(26) ? "Found" : "Not Found") << endl;
    cout << (ht.contains(99) ? "Found" : "Not Found") << endl;
    return 0;
}

=== 知识点ID: 26
=== 知识点名称: B树和B+树

## OPERATION
标题: B树查找
内容:
B树是一种多路平衡查找树，一个结点可以存放多个有序关键字和多个子树指针。
在结点内部先按关键字顺序查找目标所在范围，若命中则查找成功；否则沿对应子树继续向下查找。
所有叶子结点位于同一层，树高较低，适合磁盘、数据库索引等块式存储场景。

标题: B+树查找
内容:
B+树是 B树 的变形，内部结点主要保存索引关键字，所有数据记录通常都保存在叶子结点。
查找总是从根结点一路走到叶子结点，再在叶子结点中确认目标关键字。
叶子结点按关键字顺序通过指针连接，便于范围查询和顺序扫描。

## COMPLEXITY
标题: B树与B+树复杂度
内容:
设每个结点最多有 m 个孩子，元素总数为 n，则树高约为 O(log_m n)。
查找、插入、删除通常都需要沿根到叶的一条路径进行，时间复杂度为 O(log_m n)。
由于一个结点可以对应一个磁盘页，B树和B+树能显著减少外存访问次数。
B+树在范围查询中更有优势，因为叶子结点顺序相连。

## CODE
标题: B+树叶子层范围扫描示意
描述: 用链式叶子结点模拟 B+树适合范围查询的特点
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

struct LeafNode {
    vector<int> keys;
    LeafNode* next;
    LeafNode(vector<int> values) : keys(values), next(nullptr) {}
};

void rangeQuery(LeafNode* head, int left, int right) {
    bool first = true;
    for (LeafNode* cur = head; cur != nullptr; cur = cur->next) {
        for (int key : cur->keys) {
            if (key >= left && key <= right) {
                if (!first) cout << " ";
                cout << key;
                first = false;
            }
            if (key > right) return;
        }
    }
}

int main() {
    LeafNode a({3, 8, 12});
    LeafNode b({18, 21, 29});
    LeafNode c({35, 40, 48});
    a.next = &b;
    b.next = &c;
    rangeQuery(&a, 10, 40);
    cout << endl;
    return 0;
}
