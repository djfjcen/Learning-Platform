=== 知识点ID: 23
=== 知识点名称: 顺序查找

## OPERATION
标题: 查找的基本概念
内容:
查找（Search）是在数据集合中定位满足条件的数据元素的过程，核心问题是：给定关键字 key，判断其是否存在，或返回其位置。
查找结果分为两类：查找成功（找到目标，返回下标或指针）；查找失败（未找到，常返回 -1 或特殊标记）。
按数据是否变化分为：静态查找（只查不改，如字典检索）；动态查找（查找同时伴随插入、删除，如通讯录维护）。
查找是检索、索引、推荐、登录校验等功能的共同基础。

标题: 平均查找长度 ASL
内容:
平均查找长度（Average Search Length, ASL）是衡量查找算法效率的重要指标，表示查找过程中关键字比较次数的期望值。
对成功查找：ASL_success = Σ P_i × C_i，其中 P_i 为第 i 个元素被查找的概率，C_i 为找到该元素所需的比较次数。
对失败查找：需统计查找不命中时的比较次数期望。
ASL 越小，平均性能越好；分析时应区分成功与失败两种情形。

标题: 顺序查找基本思想
内容:
顺序查找（Sequential Search）从表的一端开始，将当前元素的关键字与目标关键字逐个比较。
若相等则查找成功并返回位置；若当前元素不相等，则继续检查下一个元素。
若扫描到表尾仍未命中，则查找失败。
该算法对存储结构要求低，数组、链表均可；对数据是否有序没有硬性要求，实现最直观。
示例：在花名册上从上到下点名找人。

标题: 哨兵优化
内容:
在数组实现中，可在表尾设置哨兵：将目标值复制到数组末尾额外位置（或利用原表最后一格），循环中只需比较元素是否等于目标，无需每次判断下标是否越界。
当目标存在于表中时，比较会在命中处结束；当目标不存在时，比较会在哨兵处结束。
哨兵法减少分支判断次数，常数因子更小，但需额外一个存储单元。

标题: 有序表上的提前终止
内容:
若顺序表按关键字有序，当当前元素已大于目标值时，后续元素更大，不可能再命中，可提前结束查找。
这是顺序查找在有序数据上的一种小优化，但仍不如二分查找高效。
链表因不支持随机访问，通常只能从头结点逐个后移，天然适配顺序扫描。

## COMPLEXITY
标题: 各类查找算法复杂度对比
内容:
| 方法 | 平均时间 | 最坏时间 | 空间 | 前提条件 |
|------|----------|----------|------|----------|
| 顺序查找 | O(n) | O(n) | O(1) | 几乎无 |
| 二分查找 | O(log n) | O(log n) | O(1) | 有序或单调 |
| 哈希查找 | O(1)* | O(n) | O(n) | 哈希函数、冲突处理 |
| BST 查找 | O(log n) | O(n) | O(n) | 树结构 |
| AVL 查找 | O(log n) | O(log n) | O(n) | 平衡维护 |
*哈希为平均复杂度，冲突严重时退化。

标题: 顺序查找性能分析
内容:
设表长为 n：
最好情况：目标在第一个位置，比较 1 次，O(1)。
最坏情况：目标不存在或位于末尾，比较 n 次，O(n)。
平均情况：等概率假设下，成功查找平均约 n/2 次比较；失败需 n 次或 n+1 次。
空间复杂度：O(1)，仅需常数个辅助变量。
适用场景：数据量小、数据无序、只偶尔查一次、实现成本优先。

## CODE
标题: 查找结果约定示例
描述: 统一成功/失败返回值，便于上层调用
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

// 返回首个等于 target 的下标，失败返回 -1
int findFirst(const vector<int>& a, int target) {
    for (size_t i = 0; i < a.size(); ++i)
        if (a[i] == target) return (int)i;
    return -1;
}

int main() {
    vector<int> data = {3, 1, 4, 1, 5};
    int idx = findFirst(data, 4);
    if (idx >= 0)
        cout << "找到，下标=" << idx << endl;
    else
        cout << "未找到" << endl;
    return 0;
}

标题: 顺序查找（数组与哨兵）
描述: 基本实现及哨兵写法
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

int sequentialSearch(const vector<int>& a, int target) {
    for (size_t i = 0; i < a.size(); ++i)
        if (a[i] == target) return (int)i;
    return -1;
}

int sequentialSearchSentinel(vector<int> a, int target) {
    a.push_back(target);  // 哨兵
    int i = 0;
    while (a[i] != target) ++i;
    if (i == (int)a.size() - 1) return -1;  // 仅命中哨兵
    return i;
}

int main() {
    vector<int> arr = {5, 2, 8, 1, 9};
    cout << sequentialSearch(arr, 8) << endl;
    cout << sequentialSearchSentinel(arr, 3) << endl;
    return 0;
}

=== 知识点ID: 24
=== 知识点名称: 二分查找

## OPERATION
标题: 二分查找基本思想
内容:
二分查找（Binary Search）要求查找区间内的数据按关键字有序（或满足单调性）。
每次取区间中点元素 mid 与目标比较：若相等则成功；若目标更小，则在左半区间继续；若更大，则在右半区间继续。
每轮比较后区间长度约减半，最多约 log₂n 轮即可确定结果。
示例：在按学号排序的名单中查找某学号。

标题: 区间与边界
内容:
实现前需统一区间含义，常见两种：
左闭右闭 [left, right]：循环条件 left <= right，mid = left + (right - left) / 2。
左闭右开 [left, right)：循环条件 left < right。
mid 建议用 left + (right - left) / 2，避免 (left + right) 整数溢出。
未命中时，可返回 -1，或返回“应插入位置”以维护有序表。

标题: 二分查找变体
内容:
在重复元素中常需找边界而非任意一次命中：
首个等于 target：找最左可行位置（lower_bound 思想）。
最后等于 target：找最右可行位置。
首个大于等于 target / 最后小于等于 target：用于分段、分数线、排名等场景。
“二分答案”：当问题具有单调可行性时，对答案空间二分而非对下标二分。

## COMPLEXITY
标题: 二分查找性能与常见错误
内容:
时间复杂度：O(log n)；空间复杂度：迭代实现 O(1)，递归实现 O(log n) 栈空间。
常见错误：数据未排序仍使用二分；left/right 更新遗漏导致死循环；边界 off-by-one；mid 溢出。
调试时可打印每轮的 left、right、mid 验证区间收缩是否正确。

## CODE
标题: 二分查找（左闭右闭）
描述: 在有序数组中查找目标，返回下标或 -1
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

int binarySearch(const vector<int>& a, int target) {
    int left = 0, right = (int)a.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (a[mid] == target) return mid;
        if (a[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}

int main() {
    vector<int> sorted = {1, 3, 5, 7, 9, 11};
    cout << binarySearch(sorted, 7) << endl;  // 3
    cout << binarySearch(sorted, 4) << endl;  // -1
    return 0;
}

=== 知识点ID: 25
=== 知识点名称: 哈希表

## OPERATION
标题: 哈希查找基本思想
内容:
哈希查找通过哈希函数 H(key) 将关键字映射到存储地址，理想情况下可直接定位，无需逐个比较。
核心步骤：计算地址 → 访问槽位 → 若冲突则按策略继续探测。
平均时间接近 O(1)，属于“空间换时间”的典型方案。
示例：用户名查账户、缓存 key 查 value、单词查释义。

标题: 哈希函数与冲突
内容:
好的哈希函数应计算快、分布尽量均匀。常用方法：除留余数法（key mod m）、乘法散列、字符串滚动哈希等。
冲突（Collision）指不同关键字映射到同一槽位，必须处理否则无法正确查找。
开放定址法：在表内按探测序列找下一个空位（线性探测、二次探测、双重散列）。
链地址法：每个槽位挂链表存放所有冲突元素，删除较方便。
再哈希法：冲突时用第二个哈希函数重新计算地址。

标题: 装填因子与扩容
内容:
装填因子 α = 元素个数 / 表长，α 越大冲突越多，查找越慢。
开放定址法在 α 接近 1 时性能急剧下降，常设阈值（如 0.7~0.75）触发扩容。
扩容需重新计算所有元素位置（重哈希），一次性成本高，但均摊后仍高效。
开放定址删除常用“墓碑”标记，避免探测链断裂。

## COMPLEXITY
标题: 哈希查找性能分析
内容:
平均情况：冲突较少时，查找、插入、删除均可视为 O(1)。
最坏情况：大量冲突或恶意构造数据时退化为 O(n)。
链地址法删除友好；开放定址法节省指针但删除与聚集问题更复杂。
与树表对比：哈希不保证关键字有序，不适合范围遍历；树表支持有序遍历与区间查询。

## CODE
标题: 链地址法哈希查找
描述: 简化版拉链法哈希表
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

class HashMap {
    vector<list<pair<int, string>>> table;
    int capacity;
    size_t hashKey(int key) const {
        return ((size_t)key * 2654435761u) % capacity;
    }
public:
    explicit HashMap(int cap = 101) : capacity(cap), table(cap) {}

    bool insert(int key, const string& val) {
        auto& bucket = table[hashKey(key)];
        for (auto& p : bucket)
            if (p.first == key) { p.second = val; return true; }
        bucket.emplace_back(key, val);
        return true;
    }

    string* find(int key) {
        for (auto& p : table[hashKey(key)])
            if (p.first == key) return &p.second;
        return nullptr;
    }
};

int main() {
    HashMap mp;
    mp.insert(1001, "Alice");
    if (auto* p = mp.find(1001)) cout << *p << endl;
    return 0;
}

=== 知识点ID: 26
=== 知识点名称: B树和B+树

## OPERATION
标题: 二叉排序树查找
内容:
二叉排序树（BST）满足：左子树所有关键字 < 根 < 右子树所有关键字。
查找从根开始：若目标等于根，成功；若更小进入左子树，否则进入右子树，直到命中或走到空指针。
插入、删除后仍需保持 BST 性质；按有序序列依次插入可能使树退化为链表，最坏查找 O(n)。
树高越低，查找路径越短，平均性能越好。

标题: 平衡二叉树（AVL）查找
内容:
AVL 树在 BST 基础上要求任意节点左右子树高度差（平衡因子）的绝对值不超过 1。
插入或删除后若失衡，通过旋转恢复平衡：LL、RR 单旋；LR、RL 双旋。
平衡后树高为 O(log n)，查找、插入、删除的最坏时间均为 O(log n)，性能更稳定。
适用于频繁增删且仍需高效查找的场景，如动态有序索引。

标题: B 树与 B+ 树查找
内容:
B 树是多路平衡查找树，结点可包含多个关键字，适合磁盘等块存储：减少 I/O 次数。
B+ 树非叶结点仅作索引，数据记录集中在叶结点，叶结点常链成有序链表，便于范围扫描。
数据库索引（如 InnoDB）广泛采用 B+ 树，在大量数据下保持 O(log n) 级查找。
与内存中的 BST/AVL 相比，B+ 树针对“块读”优化，是外存查找的典型结构。

标题: 选型口诀与决策因素
内容:
实用选型可参考：
无序、数据量小、偶尔查一次 → 顺序查找；
有序、静态、少改动 → 二分查找；
高频精确键查询、不要求有序 → 哈希查找；
频繁增删且需有序遍历或区间查询 → BST / AVL / B+ 树等树表。
决策因素包括：数据规模 n、是否有序、更新频率、内存限制、是否需要范围查询、实现与维护成本。

标题: 典型业务场景
内容:
顺序查找：短待办列表、小规模日志片段关键字扫描。
二分查找：排序词典、成绩达标线、静态排行榜名次。
哈希查找：登录用户名查账户、缓存、字典映射。
树表查找：动态价格索引、订单按时间/金额区间检索；数据库主键索引常用 B+ 树。
实际系统常组合多种结构：如先哈希定位用户，再在有序子集上二分筛范围。

## COMPLEXITY
标题: 树表查找复杂度
内容:
BST 平均查找长度与树高 h 相关，比较次数约为 h；AVL 保证 h = O(log n)。
查找复杂度与树高线性相关，与结点总数 n 的关系通过 h 体现：平衡时 O(log n)，退化时 O(n)。
B 树/B+ 树通过增大结点扇出降低树高，磁盘场景下以 I/O 次数衡量效率。

标题: 综合对比与权衡
内容:
没有“绝对最优”的查找算法，只有“最适合当前数据特征”的选择。
哈希平均最快但无序且依赖冲突处理质量；二分实现简单但要求有序；树表兼顾动态与有序但实现复杂。
评估时除渐近复杂度外，还应考虑常数因子、缓存友好性、工程可维护性与最坏情况是否可接受。

## CODE
标题: BST 查找（递归）
描述: 在二叉排序树中查找关键字
语言: C++
代码:
#include <bits/stdc++.h>
using namespace std;

struct Node {
    int key;
    Node *left, *right;
    explicit Node(int k) : key(k), left(nullptr), right(nullptr) {}
};

Node* bstSearch(Node* root, int target) {
    if (!root) return nullptr;
    if (target == root->key) return root;
    if (target < root->key) return bstSearch(root->left, target);
    return bstSearch(root->right, target);
}

int main() {
    Node* root = new Node(8);
    root->left = new Node(3);
    root->right = new Node(10);
    root->left->right = new Node(6);
    Node* hit = bstSearch(root, 6);
    cout << (hit ? "找到" : "未找到") << endl;
    return 0;
}

标题: 按场景选择查找方式（示意）
描述: 根据数据是否有序与规模选择不同策略
语言: C++

#include <bits/stdc++.h>
using namespace std;

int searchSmallUnsorted(const vector<int>& a, int x) {
    for (size_t i = 0; i < a.size(); ++i)
        if (a[i] == x) return (int)i;
    return -1;
}

int searchSorted(const vector<int>& a, int x) {
    auto it = lower_bound(a.begin(), a.end(), x);
    if (it != a.end() && *it == x) return (int)(it - a.begin());
    return -1;
}

int main() {
    vector<int> small = {9, 2, 7};
    vector<int> bigSorted = {1, 2, 3, 5, 8, 13, 21};
    cout << searchSmallUnsorted(small, 7) << endl;
    cout << searchSorted(bigSorted, 8) << endl;
    return 0;
}
